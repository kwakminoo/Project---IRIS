# Iris Domain Design

## 1. Overview

This document defines the core domain concepts of Iris.

Iris has seven main domains:

1. Assistant
2. Command
3. Automation
4. Mode
5. Monitoring
6. Safety
7. Storage

---

## 2. Core Domain Objects

## UserCommand

Represents a user input from text or voice.

Fields:
- id
- source: text | voice
- raw_text
- normalized_text
- created_at
- context

Examples:
- "작업 시작할게"
- "게임할래"
- "터미널 승인해줘"
- "GPT 답변 끝났어?"

---

## AssistantResponse

Represents Iris's response to the user.

Fields:
- id
- text
- should_speak
- response_type: chat | confirmation | alert | error
- created_at

Examples:
- "어떤 작업을 실행할까요?"
- "터미널이 승인 대기 중입니다. y를 입력할까요?"

---

## CommandIntent

Represents classified user intent.

Types:
- GENERAL_CHAT
- APP_LAUNCH
- WINDOW_CONTROL
- WEB_SEARCH
- REPORT_GENERATION
- WORK_MODE
- GAME_MODE
- CREATIVE_MODE
- MONITORING_STATUS
- COMPUTER_ACTION
- ALERT_RESPONSE
- UNKNOWN

---

## ActionRequest

Represents a computer action Iris wants to perform.

Fields:
- id
- action_type
- target
- command
- risk_level
- requires_approval
- created_at

Examples:
- Launch Cursor
- Focus Terminal
- Type "y" into terminal
- Open Chrome tab
- Arrange windows

---

## UserApproval

Represents user confirmation for an action.

Fields:
- id
- action_request_id
- approved: true | false
- approved_at
- approval_text

Rule:
No computer action can be executed without approval.

---

## ExecutionLog

Represents the result of an executed action.

Fields:
- id
- action_request_id
- result: success | failed | blocked
- message
- timestamp

---

## RecentWork

Represents a recent work session.

Fields:
- id
- title
- work_type: development | document | presentation | creative | research | game
- apps
- layout
- related_paths
- last_opened_at
- notes

Examples:
- Iris 개발 프로젝트
- AI 루키 제안서 작성
- PPT 발표 자료 제작

---

## PresetMode

Represents a dynamic mode such as work, game, or creative mode.

Fields:
- id
- mode_type: work | game | creative
- title
- suggested_apps
- suggested_layout
- requires_user_confirmation

Important:
Preset modes must ask the user before execution.

---

## MonitoredTarget

Represents a target Iris monitors.

Fields:
- id
- type: current_screen | desktop_window | browser_tab | terminal_command | system_log
- title
- process_name
- url
- handle
- enabled
- status
- last_checked_at
- last_event

Examples:
- Cursor window
- Windows Terminal
- ChatGPT Chrome tab
- Gemini Chrome tab
- Discord/Midjourney tab

---

## MonitoringEvent

Represents an event detected from a monitored target.

Fields:
- id
- target_id
- category
- confidence
- reason
- recommended_action
- created_at

Categories:
- NORMAL
- APPROVAL_WAITING
- ERROR_DETECTED
- GENERATION_FAILED
- TASK_STALLED
- RESPONSE_READY
- BUILD_NOT_STARTED
- USER_ACTION_REQUIRED
- UNKNOWN

---

## DetectionResult

Represents the output of state detection.

Fields:
- category
- confidence
- reason
- recommended_action

Example:
Input:
"Proceed? (y/n)"

Output:
category: APPROVAL_WAITING
confidence: 0.95
reason: Terminal is waiting for user approval.
recommended_action: Ask the user whether to type y.

---

## SafetyPolicy

Represents safety rules for blocking dangerous actions.

Blocked actions:
- File deletion
- Payment
- Password input
- Personal data submission
- System setting changes
- Dangerous shell commands
- Unapproved keyboard/mouse control

---

## WebReport

Represents a report generated by the web agent.

Fields:
- id
- query
- title
- summary
- sources
- key_points
- created_at
